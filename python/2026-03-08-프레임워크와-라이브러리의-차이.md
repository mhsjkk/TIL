
### ### Context
*   사용자가 소프트웨어 개발에서 자주 접하는 '프레임워크'와 '라이브러리'의 개념적 차이에 대한 이해를 돕기 위함입니다. 두 용어는 종종 혼용되지만, 소프트웨어 설계 및 개발 흐름에서 중요한 차이를 가집니다.

### ### Core
*   **프레임워크 (Framework)**
    *   **정의**: 애플리케이션의 전체적인 구조나 골격(skeleton)을 제공하는 소프트웨어 환경입니다. 특정 유형의 애플리케이션을 개발하기 위한 기반을 제공하며, 개발자가 따라야 할 규칙과 흐름을 정의합니다.
    *   **제어의 역전 (Inversion of Control, IoC)**: 프레임워크의 가장 큰 특징 중 하나는 '제어의 역전'입니다. 개발자가 작성한 코드가 프레임워크에 의해 호출되고 실행됩니다. 즉, 프레임워크가 애플리케이션의 흐름을 제어하고, 개발자는 프레임워크의 특정 지점(hooks)에 필요한 로직을 채워 넣습니다.
    *   **예시**: 웹 개발 프레임워크 (Django, Spring, Ruby on Rails), 모바일 앱 개발 프레임워크 (Flutter, React Native).
*   **라이브러리 (Library)**
    *   **정의**: 특정 기능을 수행하는 데 필요한 함수, 클래스, 모듈 등의 집합입니다. 개발자가 자신의 코드에 가져와(import) 필요한 시점에 직접 호출하여 사용합니다.
    *   **제어권**: 라이브러리는 개발자에게 제어권이 있습니다. 개발자가 원하는 시점에 라이브러리의 기능을 호출하고 결과를 사용합니다. 라이브러리는 전체 애플리케이션의 흐름에 관여하지 않고, 특정 문제 해결을 위한 도구 역할을 합니다.
    *   **예시**: 데이터 처리 라이브러리 (NumPy, Pandas), UI 라이브러리 (jQuery, React), 이미지 처리 라이브러리 (Pillow).

### ### Insight
*   프레임워크와 라이브러리의 핵심적인 차이는 '제어권(Control)'에 있습니다. 라이브러리는 개발자가 필요할 때 호출하는 도구인 반면, 프레임워크는 애플리케이션의 전반적인 구조와 흐름을 정의하고 개발자의 코드를 호출하여 실행하는 '틀'입니다. 프레임워크는 일관된 구조와 빠른 개발을 돕지만, 프레임워크의 규칙을 따라야 한다는 제약이 있습니다. 반면 라이브러리는 유연하게 필요한 기능만 가져다 쓸 수 있지만, 전체 애플리케이션의 구조는 개발자가 직접 설계해야 합니다.
