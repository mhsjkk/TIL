### Context
Rust 프로그래밍 언어의 주요 특징을 이해하고, 이를 통해 Rust가 가진 장점을 파악하고자 한다.

### Core
Rust는 다음과 같은 특징을 가진다.

*   **메모리 안전성 (Memory Safety) 및 가비지 컬렉션 없음 (without Garbage Collection)**: Rust는 컴파일 시간에 메모리 안전성을 검사하여 런타임 오버헤드 없이 안전한 메모리 관리를 제공한다.
*   **소유권 시스템 (Ownership System)**: 각 값은 하나의 소유자만 가질 수 있으며, 소유자가 범위를 벗어나면 메모리가 자동으로 해제된다.
*   **빌림과 참조 (Borrowing and References)**: 소유권을 옮기지 않고 데이터에 접근할 수 있도록 빌림 규칙을 통해 데이터 경쟁을 방지한다.
*   **패턴 매칭과 열거형 (Pattern Matching and Enums)**: 강력한 패턴 매칭 기능을 통해 데이터의 다양한 형태를 안전하고 효율적으로 처리할 수 있다.
*   **에러 핸들링 (Error Handling)**: Result 타입을 사용하여 에러를 명시적으로 처리하도록 강제하여 안정성을 높인다.

### Insight
Rust의 특징들은 안전하고 효율적인 시스템 프로그래밍을 가능하게 한다. 특히, 가비지 컬렉터 없이 메모리 안전성을 보장하는 점은 성능이 중요한 영역에서 큰 장점이다. 복잡한 소유권 시스템은 학습 곡선을 높이지만, 결과적으로 런타임 에러를 줄이고 예측 가능한 코드를 작성하는 데 도움을 준다.
