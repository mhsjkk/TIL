### Context
하스켈 언어의 특징을 이해하기 위한 배경 정보입니다. 하스켈은 함수형 프로그래밍 언어로서 순수성, 타입 시스템, 그리고 고급 추상화 기능을 특징으로 가지고 있습니다. 이 문서에서는 하스켈의 주요 특징을 기술적 정확성을 유지하면서 설명합니다.

### Core
- **순수성 (Purity)**: 하스켈은 함수가 부작용(side effect)을 일으키지 않으며, 입력 인수만으로 결과를 계산합니다. 함수는 항상 동일한 입력에 대해 동일한 출력을 반환합니다.
- **강력한 타입 시스템 (Strong Static Typing)**: 하스켈은 타입 추론(type inference)을 지원하며, 모든 표현식의 타입을 컴파일 시점에 검증합니다. 타입 시스템은 하스켈의 핵심 특징 중 하나입니다.
- **느긋한 평가 (Lazy Evaluation)**: 하스켈은 필요할 때까지 계산을 늦춥니다. 이는 무한 자료구조(infinite data structures)를 효율적으로 처리할 수 있도록 도와줍니다.
- **모나드 (Monad)**: 모나드는 하스켈에서 부작용을 관리하는 방법입니다. 예를 들어, I/O 작업이나 상태 변이는 모나드를 통해 순수한 함수 체인으로 표현됩니다.
- **패턴 매칭 (Pattern Matching)**: 하스켈은 데이터 구조를 쉽게 분석할 수 있도록 패턴 매칭을 지원합니다. 함수 정의에서 패턴 매칭을 사용하여 다양한 경우를 처리할 수 있습니다.

### Insight
하스켈의 순수성과 타입 시스템은 코드의 안정성을 높여 버그를 줄이고, 모듈화를 강화합니다. 느긋한 평가는 효율적인 자료구조 처리를 가능하게 하지만, 학습 곡선이 높습니다. 모나드는 복잡한 부작용 관리를 순수한 함수 체인으로 추상화하지만, 이를 이해하는 데 시간이 필요합니다. 하스켈은 연구 및 학술 분야에서 널리 사용되며, 현대 함수형 언어의 기반이 되었습니다.
