### Context
프로그래밍 초보자로서 객체지향 프로그래밍(OOP)의 개념이 추상적으로 느껴져 실제 코드에서 어떻게 활용되는지 이해하기 어려웠습니다. 단순한 함수와 변수만으로도 프로그램을 작성할 수 있지만, 프로그램 규모가 커질수록 코드 관리가 복잡해지는 문제를 경험했습니다.

### Core
```python
# 절차적 프로그래밍 방식
car1_name = "소나타"
car1_speed = 0
car1_color = "빨강"

car2_name = "그랜저" 
car2_speed = 0
car2_color = "파랑"

def car_accelerate(car_speed):
    return car_speed + 10

def car_brake(car_speed):
    return car_speed - 10

# 객체지향 프로그래밍 방식
class Car:
    def __init__(self, name, color):
        self.name = name
        self.speed = 0
        self.color = color
    
    def accelerate(self):
        self.speed += 10
        return self.speed
    
    def brake(self):
        self.speed -= 10
        return self.speed

# 객체 생성 및 사용
sonata = Car("소나타", "빨강")
granger = Car("그랜저", "파랑")

sonata.accelerate()  # 속도 10 증가
granger.brake()      # 속도 10 감소
```

### Insight
객체지향 프로그래밍의 가장 큰 장점은 데이터와 기능을 하나의 단위로 묶어 관리하는 것입니다. Car 클래스는 speed, color, name 속성을 가지며 accelerate(), brake() 메서드를 제공합니다. 이렇게 캡슐화하면 코드 재사용성과 유지보수성이 크게 향상됩니다.

* **캡슐화**: 데이터(속성)와 기능(메서드)를 하나의 클래스로 묶음
* **재사용성**: 동일한 클래스로 여러 객체 생성 가능
* **확장성**: 상속을 통해 기존 클래스 확장 가능
