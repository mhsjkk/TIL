### Context
웹서비스의 개발과 운영(部署)에는 다양한 도구가 필요합니다. 프론트엔드, 백엔드, 데이터베이스, 배포, 모니터링 등 각 계층별로 적합한 도구를 선택하고 조합해야 안정적인 서비스를 구축할 수 있습니다. 최신 개발 환경에서는 컨테이너 기반 배포와 클라우드 네이티브 아키텍처가 일반화되어 관련 도구 사용이 필수적입니다.

### Core
#### 프론트엔드 개발
* React, Vue, Angular와 같은 메이저 프레임워크
* Next.js, Nuxt.js와 같은 메타프레임워크(SSR/SSG 지원)
* 상태 관리: Redux, Vuex, Pinia, Zustand
* 빌드 도구: Vite, Webpack, esbuild
* 패키지 매니저: npm, yarn, pnpm

#### 백엔드 개발
* 런타임: Node.js, Python, Java, Go
* 프레임워크: Express, Django, Spring Boot, FastAPI, Ruby on Rails
* API: REST, GraphQL(예: Apollo, Relay), gRPC
* 인증/인가: OAuth2, JWT, Passport.js, Spring Security

#### 데이터베이스
* 관계형: PostgreSQL, MySQL, MariaDB
* 문서형: MongoDB
* 키-값: Redis, Memcached
* 검색: Elasticsearch

#### 배포 및 인프라
* 컨테이너: Docker, Podman
* 오케스트레이션: Kubernetes, Docker Swarm
* IaC(Infrastructure as Code): Terraform, AWS CloudFormation
* 서버리스: AWS Lambda, Vercel, Netlify
* 클라우드: AWS, GCP, Azure

#### CI/CD
* GitHub Actions, GitLab CI, Jenkins, CircleCI
* 아티팩트 저장: Nexus, Artifactory

#### 모니터링 및 로깅
* 모니터링: Prometheus, Grafana, Datadog, New Relic
* 로깅: ELK Stack(Elasticsearch, Logstash, Kibana), Loki
* 분산 추적: Jaeger, Zipkin

#### 개발 지원
* 버전 관리: Git
* 코드 품질: ESLint, Prettier, SonarQube
* 테스트: Jest, Pytest, JUnit, Cypress, Playwright
* 컨피그 관리: Ansible, Chef, Puppet
* 메시지 큐: RabbitMQ, Apache Kafka

### Insight
웹서비스 개발 도구 선택은 프로젝트 규모, 팀 스택, 운영 환경에 따라 달라집니다. 최신 트렌드는 모놀리식에서 마이크로서비스로, 그리고 서버리스 아키텍처로 이동하고 있습니다. 또한 AI 기반 개발 도구(예: GitHub Copilot)가 생산성을 높이고 있습니다. 컨테이너와 쿠버네티스는 이제 표준 배포 플랫폼으로 자리잡았으며, 관찰 가능성(Observability)을 위한 모니터링 도구 통합이 중요해졌습니다. 
도구의 수가 너무 많아지면 복잡성이 증가하므로, 필요 최소한의 도구로 시작하고 점진적으로 확장하는 전략이 권장됩니다.
